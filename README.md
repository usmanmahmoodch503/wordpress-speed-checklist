# WordPress Speed Checklist

A simple checklist and notes for improving WordPress speed and overall website performance.

## What I check first

- Hosting quality
- Theme performance
- Plugin load
- Page size
- Image optimization
- Caching setup
- Mobile speed
- Core Web Vitals

## Basic WordPress speed checklist

### 1. Hosting
- Check if hosting is slow or overloaded
- Test server response time
- Make sure PHP version is updated

### 2. Theme
- Use a lightweight and well-coded theme
- Remove unnecessary theme features
- Avoid heavy animations where not needed

### 3. Plugins
- Remove unused plugins
- Replace heavy plugins with lightweight alternatives
- Check if too many plugins are loading scripts on every page

### 4. Images
- Compress large images
- Use proper image sizes
- Prefer WebP where possible
- Avoid uploading very large files

### 5. Caching
- Enable page caching
- Enable browser caching
- Use a caching plugin if needed

### 6. CSS and JavaScript
- Minify CSS and JavaScript files
- Delay or defer non-critical JavaScript
- Remove unused CSS where possible

### 7. Database
- Clean post revisions
- Remove spam comments
- Delete old transients
- Optimize database tables

### 8. CDN
- Use a CDN for faster global delivery
- Check if static files are loading correctly

### 9. Mobile performance
- Test website speed on mobile devices
- Reduce heavy sections above the fold
- Optimize sliders, videos, and popups

### 10. Final testing
- Test homepage, service pages, and product pages
- Compare before and after results
- Recheck user experience after changes

## Tools often used

- PageSpeed Insights
- GTmetrix
- Google Search Console
- LiteSpeed Cache
- WP Rocket
- ShortPixel

## Goal

The main goal is to make a WordPress website faster, cleaner, and better for users.
